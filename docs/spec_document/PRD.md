## 1. 项目概述

**产品名称**：BBQ翻译（Borderless Books Quickly）

**构建什么**：一款 Windows 桌面应用程序，内置 AI 翻译管线，让用户能够将漫画/图片中的文字（日语为主，兼容中英韩）自动翻译为目标语言，并提供可视化编辑与导出能力。

**为谁构建**：
- **核心用户**：无 NVIDIA 独立显卡、内存 8–16 GB、希望"开箱即用"的非开发者。
- **次要用户**：有一定技术背景、希望接入自定义 API 或安装高级拓展模块的进阶用户。

**核心价值**：
- 一键翻译漫画，无需配置 Python 环境或命令行操作。
- 内置轻量模型（CPU 可运行），不依赖独立显卡。
- 按需安装高级模型拓展（GPU 加速、高精度 OCR 等），保持初始安装包精简。

## 2. 核心功能（In-Scope）

### 2.1 翻译管线 ✅
| 功能 | 说明 |
|------|------|
| **文字检测** | 自动检测漫画气泡中文字区域（CTD / DBConvNext 检测器） |
| **OCR 识别** | 识别检测到的文字（支持 32px / 48px / 48px_ctc / MangaOCR） |
| **在线翻译** | 调用第三方 API 翻译文字（DeepSeek / Gemini / ChatGPT / Groq / Custom OpenAI / 百度 / 有道等） |
| **去字修复** | 擦除原文后填充背景（LaMa MPE / LaMa Large） |
| **文字渲染** | 将译文嵌入图片（默认渲染器 / manga2eng，内置字体） |

### 2.2 批量翻译 ✅
- 支持拖拽或选择**单图、多图、压缩包（ZIP/CBZ/CBR）、PDF、EPUB**。
- 翻译进度实时展示，支持单任务取消。
- 翻译结果写入 Book/Chapter/Page 数据结构并持久化到书架。

### 2.3 翻译编辑器 ✅
- **双面板预览**：左侧显示原图，右侧显示翻译结果。
- **文字区域可视化**：Konva Canvas 在图片上绘制所有识别到的文字框。
- **单框编辑**：点击文字框，在右侧面板修改译文、字体、字号、方向、颜色等。
- **手动框选**：用户可手动绘制新文字框并触发 OCR + 翻译。
- **重新修复**：对选定区域重新执行 Inpainting。
- **重新渲染**：保存编辑后的所有区域，重新渲染整页。
- **多页导航**：在章节内按页切换。

### 2.4 书架管理 ✅
- **书库列表**：展示所有书籍（MangaBook），支持创建、编辑标题、删除、设置封面。
- **快速翻译内容**：系统内置 `quick-book`，临时翻译内容默认存入此书。
- **书籍详情**：展示章节列表（MangaChapter），支持批量翻译、删除、导出章节。
- **章节阅读**：进入阅读器或编辑器查看/修改章节内容。

### 2.5 阅读器 ✅
- 滚动模式（纵向连续滚动）和翻页模式（逐页查看）。
- 支持键盘方向键翻页、鼠标滚轮缩放。
- 可在阅读器和编辑器之间切换。

### 2.6 拓展中心 ✅
- 展示所有可用拓展模块及其安装状态、体积、描述。
- 支持在线下载安装（国内 HF Mirror 直连）。
- 支持离线导入（本地 `.zip` / `.whl` 文件）。
- 支持卸载拓展。
- 实时展示安装进度（下载速度、百分比）。

### 2.7 API 凭证管理 ✅
- 在设置弹窗中配置各翻译服务的 API Key。
- 支持 DeepSeek / Gemini / Custom OpenAI（Ollama 等） / Groq。
- Key 存储在前端 `localStorage`，通过请求头传递给后端，**不经过服务端持久化**。
- 支持 API Key 连通性验证（`/verify_api_key`）。

### 2.8 字体管理（API ✅ · 端到端 ⬜ 待测试）
- 查看内置字体列表。
- 上传自定义字体（`.ttf` / `.otf`）。
- 删除自定义字体。
- 编辑器中实时选择字体应用于文字渲染。

> ⚠️ **当前状态**：后端 API 端点已实现，但因缺乏测试用字体文件，端到端流程（上传→选择→渲染）尚未完成验证。

## 3. 范围外功能（Out-of-Scope for v1.0）

以下功能在 v1.0 核心包中**不包含**，均通过**拓展模块**按需安装：

| 功能 | 原因 | 归属 |
|------|------|------|
| GPU 加速推理（CUDA） | 体积 ~2.5 GB，且依赖 NVIDIA 显卡 | CUDA 拓展模块 |
| MangaOCR（高精度日语 OCR） | 模型 ~400 MB，需推荐 CUDA | MOCR 拓展模块 |
| LaMa Large（高质量去字） | 模型 ~350 MB，需推荐 CUDA | LaMa Large 拓展模块 |
| 离线翻译（NLLB / 本地 LLM） | 模型 >2 GB，必须 CUDA | 离线翻译拓展（未来规划） |
| 图片超分辨率 | 模型 ~60 MB | Upscaler 拓展（未来规划） |
| 漫画上色 | 模型 ~200 MB，需 CUDA | Colorizer 拓展（未来规划） |
| 云同步 / 多设备 | 架构复杂度 | 后续版本 |
| 自动更新（OTA） | 需要更新服务器 | 后续版本 |
| macOS / Linux 支持 | 当前仅 Windows Electron 打包 | 后续版本 |
| 暗色主题 | 前端尚未实现 | 待规划 |
| 多语言 UI（国际化） | 当前仅中文 UI | 待规划 |

## 4. 用户故事

### 用户故事 1：快速翻译单张漫画图片
> 作为一个日语零基础用户，我想拖拽一张日语漫画截图到 BBQ翻译，选择目标语言为简体中文，点击翻译按钮，在 10 秒内看到带有中文译文的图片，这样我就能理解漫画内容了。

### 用户故事 2：批量翻译漫画章节
> 作为一个经常阅读生肉漫画的用户，我想一次性导入一个 ZIP 压缩包（包含 50 张漫画页），点击批量翻译，等待期间查看逐页进度，翻译完成后在书架里找到这一章并进行阅读。

### 用户故事 3：编辑并修正翻译
> 作为一个追求翻译质量的用户，我想在编辑器里点击某个翻译错误的文字框，手动修改译文，然后点击重新渲染，导出修改后的图片。

### 用户故事 4：安装高精度 OCR 拓展
> 作为一个有 NVIDIA 显卡的用户，我想在拓展中心看到 MangaOCR 模块的介绍和下载进度，点击安装，等待下载完成后切换 OCR 引擎到 mocr，获得更精准的日语识别效果。

### 用户故事 5：配置翻译 API
> 作为一个新用户，我在首次启动时打开设置，填入 DeepSeek API Key，点击验证通过后，回到主页开始我的第一次翻译。

## 5. 成功标准（Definition of Done）

### v1.0 核心包完成标准
- [ ] 用户能在 Windows 10/11 上通过安装包一键安装，无需预装 Python 或任何依赖。
- [ ] 冷启动时间：SSD ≤ 8 秒，HDD ≤ 15 秒（至 UI 可交互）。
- [ ] 安装包体积 ≤ 500 MB（压缩后）。
- [ ] 单张漫画图片翻译全流程（导入→翻译→渲染）不超过 15 秒（i5-12400 级别 CPU）。
- [ ] 翻译管线全链路无崩溃：单图翻译 → 批量翻译 → 编辑器编辑 → 导出章节。
- [ ] 拓展中心能成功安装、卸载至少 3 种拓展（MOCR、LaMa Large、CUDA）。
- [ ] API Key 配置、验证、翻译请求全流程无错误。

### Phase 3（打包分发）完成标准
- [ ] `electron-builder` 生成 Windows NSIS 安装包，可在全新系统上安装。
- [ ] 安装 → 启动 → 导入图片 → 翻译 → 编辑 → 导出 冒烟测试全部通过。
- [ ] 参见 `DESKTOP_PACKAGING_TEST_PLAN.md` 完整测试清单。

## 6. 非目标（明确避免的方向）

- **不做 SaaS 或云端服务**：BBQ翻译 是纯本地桌面应用，无用户账号体系。
- **不存储用户数据到云端**：所有图片、翻译结果、API Key 均保存在用户本地。
- **不构建翻译模型**：仅调用第三方翻译 API 或使用现有开源模型，不自研翻译引擎。
- **不支持实时协作**：单人本地使用，无多人协同功能。
- **不提供 Web 托管版本**：虽然代码可在浏览器运行，但 v1.0 文档仅覆盖 Windows 桌面端场景。

## 7. 依赖与约束

- **翻译质量**：依赖第三方 API（DeepSeek / Gemini 等），API 可用性和质量非本产品控制范围。
- **网络依赖**：翻译 API 必须联网；文字检测 / OCR / 去字修复均支持完全离线运行。
- **Python 运行时**：嵌入式 Python 3.11（`python-embed-amd64`），与用户系统 Python 完全隔离。
- **硬件最低要求**：内存 ≥ 8 GB，磁盘剩余 ≥ 2 GB（含拓展模块空间）。
